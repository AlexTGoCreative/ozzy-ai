# Software Bill of Materials (SBOM)

OPSWAT SBOM (Software Bill of Materials) analyzes the third-party packages in source code and containers, identifies known vulnerabilities, then generates inventories of these software components. With SBOM, users can scan track dependencies in large codebases to understand the risks associated with open-source software (OSS) and secure their applications.

To enable this engine, please go to Workflow > Workflow name > SBOM

Support:

- Source code and container
- CyclonceDX, SPDX report from external tools.
